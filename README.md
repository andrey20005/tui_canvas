1. Ядро состояния (Хранилище данных) \
    Эти структуры живут внутри Screen и управляются только основным потоком.
    - Screen: Главный фасад. Теперь владеет состоянием без мьютексов.
    - Canvas: Матрица графических данных (цвета).
    - TextLayer: Матрица текстовых данных (символы + шейдеры).
    - TerminalCell: Атомарная единица RawFrame (символ + цвета текста/фона).
1. Коммуникации (Конвейер передачи данных) \
    Эти сущности обеспечивают взаимодействие между горутинами.
    - renderChan (chan *RawFrame): Канал для передачи подготовленных кадров в рендерер. Передает владение памятью.
    - keyChan (chan KeyEvent): Канал для передачи событий клавиатуры пользователю.
    - mouseChan (chan MouseEvent): Канал для передачи событий мыши пользователю.
    - resizeChan (chan ResizeEvent): Внутренний сигнальный канал. watchResize пишет сюда, основной поток читает отсюда.
    - done (chan struct{}): Канал остановки всех фоновых горутин.
    - framePool (sync.Pool): Пул объектов RawFrame для исключения аллокаций.
1. Фоновые процессы (Горутины) \
    Каждый процесс живет в своем потоке исполнения.
    - readInput(): Бесконечно читает из os.Stdin, парсит ANSI-последовательности, конвертирует их в события и отправляет в keyChan или mouseChan.
    - watchResize(): Бесконечно ждет сигнала SIGWINCH. При получении отправляет ResizeEvent в resizeChan. Ничего не меняет в холстах.
    - renderLoop(): Бесконечно ждет кадр из renderChan. Выполняет "медленный" вывод в os.Stdout, после чего возвращает кадр в framePool.
1. Основной поток (Оркестратор) \
    Это единственный поток, имеющий право изменять Canvas, TextLayer и Screen.
    Основной цикл (Main Loop):
    1. Обработка событий: Читает resizeChan (если есть — делает Canvas.Resize() и TextLayer.Resize()).
    1. Выполнение кода пользователя: Вызывает screen.Draw(callback).
    1. Построение кадра: Получает кадр из framePool, вызывает BuildFrom (строит снимок).
    1. Передача: Отправляет кадр в renderChan (передает владение).
    1. Управление: Игнорирует попытки других горутин менять состояние, так как они не имеют к нему доступа (или сигналят через каналы).
1. Транспортные структуры -
    Данные, которые "летают" между сущностями.
    - RawFrame: "Плоский" слепок экрана (используется для передачи в рендерер).
    - KeyEvent, MouseEvent, ResizeEvent: Структуры событий для пользователя.

// TerminalCell хранит состояние одного знакоместа терминала.
type TerminalCell struct {
	Rune rune  // Символ ('▀' или буква из TextLayer)
	Fg   Color // цвет символа
	Bg   Color // цвет фона
}

// RawFrame — промежуточный плоский слепок экрана.
type RawFrame struct {
	cells  [][]TerminalCell // строчка с индексом 0 нижняя 
	columns  uint
	rows uint
